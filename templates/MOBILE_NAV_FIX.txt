===============================================
MOBILE HAMBURGER MENU FIX FOR INDEX.HTML
===============================================

STEP 1: Add this CSS code after line 25 (after the tailwind config script closes and before </head>):

<style>
/* Fix AOS conflicts with mobile navigation */
html {
    overflow-x: clip;
}

body {
    max-width: 100vw;
}

/* Ensure navigation works with AOS */
nav, nav *, #mobileMenu, #mobileMenu * {
    pointer-events: auto !important;
}

nav {
    isolation: isolate;
}

/* Ensure hamburger button is always clickable */
#mobileMenuBtn {
    position: relative;
    z-index: 9999 !important;
    pointer-events: auto !important;
    touch-action: manipulation;
}

#mobileMenuBtn span {
    pointer-events: none !important;
}

/* Ensure buttons in navigation work */
nav button {
    cursor: pointer;
}

/* Fix for mobile touch events */
@media (max-width: 1024px) {
    #mobileMenuBtn {
        -webkit-tap-highlight-color: rgba(0, 0, 0, 0.1);
    }
}
</style>

STEP 2: On line 28, change:
<body class="font-sans antialiased overflow-x-hidden">

TO:
<body class="font-sans antialiased">

STEP 3: On line 30, change:
<nav class="fixed top-0 left-0 right-0 bg-white z-50">

TO:
<nav class="fixed top-0 left-0 right-0 bg-white z-[100]">

STEP 4: On line 96, change:
class="fixed top-0 right-0 h-full w-full max-w-sm bg-white z-50 shadow-2xl...

TO:
class="fixed top-0 right-0 h-full w-full max-w-sm bg-white z-[110] shadow-2xl...

STEP 5: Save the file and test on your mobile device.
